📋 v1.4 vs v1.5 差異總覽

Aspect v1.4（現況） v1.5（這份 mockup）
結果區位置 Floating PipelineResultsPanel（pop-up，遮 dashboard） Inline，直接嵌在中央 Canvas 下方
建構期間中央區 空白 dashboard，無視覺進度 Mini Pipeline Canvas，逐步繪出 node + edge
Plan Panel 位置 飄在 chat 訊息中（NEW 標） 直接放在 Stage 進度條下方，與 console 整合
建構 ops 訊息 每個 op（add_node / rename）都是大 chip 占滿 預設摺疊「建構過程（N ops）」，點開才看明細
「編輯」link 行為 跳到 /admin/pipeline-builder/new 但被 wizard Step 1/3 攔截，畫面是空的 直接進已載入該 pipeline 的 builder canvas（繞過 wizard）
跨頁 chat 保留 進 builder 後 AIAgentPanel state 重置 → 訊息清空 chat history 跨頁存活（global store / sessionStorage）
失敗退路 只有 propose_pipeline_patch 一條（agent 自動修） 維持 patch + 紅色「自己改」link（兩條退路並存）
