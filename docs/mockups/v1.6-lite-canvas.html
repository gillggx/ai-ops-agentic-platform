🎨 AIOps v1.6 — Lite Canvas Overlay

定義： Lite Canvas = 「乾淨 DAG view」 — 沒有 Pipeline Builder 的 toolbar / block library / Parameters inspector，只有畫布、minimap、Auto Layout。
定位： Pop-up overlay 蓋住左側 sidebar + 中央 main，不蓋右側 AI Agent rail（chat 持續可用）。
切換規則：
  • 建構中 → Canvas tab；結果 tab disabled（灰）
  • build done + auto-run 跑完（成功或失敗）→ 自動切到「📊 結果」tab
  • 使用者手動點回 Canvas tab 都隨時可以；結果 tab 一旦有資料，badge 提示常駐
離開： 「× 關閉」 收起 overlay；「🛠 開啟編輯」帶 sessionStorage 跳到真正的 Pipeline Builder。

📋 v1.5 vs v1.6 差異

Aspect v1.5 reverted（剛才這版） v1.6 Lite Canvas（這份 mockup）
建構期間 LiveCanvasOverlay — 完整 BuilderLayout（block library / inspector / toolbar 全在） Lite Canvas — 只有 DAG + minimap + Auto Layout，乾淨
結果展示 浮窗 PipelineResultsPanel（與 Canvas overlay 並存，視覺重） 同 overlay 第二個 tab；切過去看，不再多開浮窗
「進真正 builder」 chat 內 markdown link，要捲到底找 overlay header 永遠有「🛠 開啟編輯」按鈕
右側 AI Agent 同樣 plan + ops 摺疊（不變） 完全不變
離開動作 ESC 或 ×；result panel 各自關 overlay 一個 ×（兩個 tab 一起關）
