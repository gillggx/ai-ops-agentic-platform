設計意圖（給 design 團隊的 brief）

區塊回答的問題資料源（既有 SSE，零後端新碼）
1. Agent 狀態列「現在誰在動？」— 三 agent 晶片，活動中的有脈動點；顯示 phase/round 與預算用量（round 2/32、replan 0/2、repair 0/3）llm_usage（帶 agent 歸因）、pb_glass_op._round、plan_confirmed
2. 分道活動流「動到哪了？」— 按 agent 上色的時間軸：Planner 提案/確認（含移除 chip）、Builder 每個動作+結果行、Verifier 裁決（ADVANCED/REJECTED）、Graph 的確定性移除pb_glass_op、phase_update、nodes_removed、goal_plan_confirmed
3. 記憶效應「這次引用/學到什麼？」— 召回清單（含「新蒸餾」徽章標 Supervisor 產出的）+ W1/W2/W3 寫入memory_recall、memory_write（前端已有 case）
4. 成本「花了多少？」— per-agent tokens + cache 命中率llm_usage、glass_usage
D6. 三段式說明每步顯示 決定/理由/依據；「依據」分兩級視覺：▣ 系統事實（實線，verifier 裁決/實測 rows/user 勾選）vs ⚠ agent 自述（虛線 — 模型講的理由，可能事後編，修正時優先質疑）；引用記憶顯示其 How-to-apply 句而非只有編號phase_update/nodes_removed（事實）+ 工具 reason 欄（新增，輕量）+ memory_recall
D7. 「教它」時刻召回 0 筆的步驟高亮 + 一鍵開 /agent-knowledge，自動帶入當下情境（block/phase/指令）；補的知識 30 秒 embed，下次 build 生效並在此可見 — 「看見 → 理解 → 補知識」閉環的入口memory_recall（空清單偵測）+ 既有知識新增 API
