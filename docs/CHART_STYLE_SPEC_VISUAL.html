01主戰場：趨勢圖 line_chart —「畫 EQP-01 STEP_001 標準 SPC 管制圖，提示顯示 lotID」

現況簡配版：只有三條線 + 紅圈
滑過點只看到「x / value」。沒有 σ 區帶、軸沒有名字、紅圈不解釋原因 — 不是標準管制圖。
期望標準 SPC：區帶 + 可解釋違規 + 指定欄位提示
Zone C/B/A 三層 σ 區帶、CL 實線、UCL/LCL 紅虛線帶值、滑過紅點顯示違反哪條 WECO 規則 + lotID/recipe、y 軸有名字有單位。
# agent 對這句指令該下的參數（粗體 = 本次新增）
{ "x": "eventTime", "y": "value",
  "ucl_column": "ucl", "lcl_column": "lcl", "center_column": "center",
  "highlight_field": "is_ooc",
  "style": { "spc_zones": true, "y_label": "xbar (nm)" },
  "tooltip_fields": ["lotID", "recipe"],
  "weco_annotate": true }

反向指令也要會 —「簡潔版管制圖，不要區帶」（STYLE-10 的 S5）

期望agent 下 style.spc_zones=false
管制線照畫、tooltip 照給，只拿掉區帶 — 測 agent 是「聽懂指令」還是「無腦開全套」。
為什麼 S5 是關鍵題：如果文件只寫「標準管制圖 = 開 spc_zones」， agent 會把每張圖都開區帶。文件必須寫成原則（「spc_zones 控制 σ 區帶顯示，預設跟隨管制線」）， agent 才能雙向操作。這正是 STYLE-10 要驗的「有沒有看懂 document」。

02X̄-R 管制圖 xbar_r —「違規點要能看出違反哪條規則」

現況WECO 紅點會標，但滑過毫無反應
這張圖現在完全沒有 tooltip（試試看 — 滑過任何點都沒有提示）。紅點知其然不知其所以然。
期望σ 區帶 + 違規原因進提示
σ 本來就算好了（WECO 判定在用），補畫區帶零成本；滑過紅點顯示「違反 R2：連續 9 點同側」這類可行動資訊。
{ "value_column": "value", "weco_rules": ["R1","R2","R3"],
  "weco_annotate": true, "style": { "spc_zones": true },
  "tooltip_fields": ["subgroup", "n"] }

03直方圖 histogram_chart —「xbar 分佈，規格線 LSL 48 / USL 52」

現況只有 bins 跟單位，沒有規格線、沒有提示

期望LSL/USL 規格線 + 常態曲線 + bin 提示
紅虛線規格界線、常態疊線看偏態、滑過每根柱顯示區間與筆數。
{ "value_column": "value", "bins": 18, "lsl": 48, "usl": 52, "normal_overlay": true }
