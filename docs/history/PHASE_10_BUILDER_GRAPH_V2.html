Phase 10 — Builder Mode Graph v2 (Graph-Heavy Refactor)

Spec date: 2026-05-09 · status: APPROVED, in development · branch: main · feature flag: AGENT_BUILD_GRAPH=v2

把 Builder Mode /agent/build 從「1 個 graph node + 80-turn 自由 LLM tool-use loop」改寫成 10-node LangGraph state machine。LLM 決策權壓縮到 3 個窄推理任務（plan / repair_plan / repair_op）， graph 控制流程、驗證、retry、confirm gate、finalize。

Graph node (deterministic — pure code)
LLM call (Haiku 4.5, narrow reasoning)
Read-only canvas access
Mutating canvas tools
Gate (interrupt — 等 user)

① 設計原則

  • LLM 不決定 flow：「下一步做什麼」永遠由 graph cursor 決定，不是 LLM 看著辦。
  • LLM 只回答 graph 問的具體問題：產 plan / 修壞掉的 plan / 修壞掉的 op args。
  • 每個 graph node 都是 pure function：可單測、可 stub LLM、可看 trace。
  • Confirm gate 是 graph node 不是 prompt 規則：強制中斷 → 等 user → resume。
  • Retry / escalation 政策寫在 graph 不在 LLM：repair_op 失敗 N 次升級到 repair_plan。
  • 跟舊 stream_agent_build 並存：env flag AGENT_BUILD_GRAPH=v2，灰度切換。

④ Op schema (一次成形 plan)

plan 是 list[Op]，每個 Op 是 5 種 type 之一。LLM 在 plan_node 一次產出整個 list，後續不允許 mid-flight 加 op（E8 決議）。

class OpType(str, Enum):
    ADD_NODE    = "add_node"     # 加 1 個 node
    CONNECT     = "connect"      # 接 2 個 node
    SET_PARAM   = "set_param"    # 改 node 的 1 個 param
    RUN_PREVIEW = "run_preview"  # 預覽 node 輸出（debug）
    REMOVE_NODE = "remove_node"  # 移除 node

class Op(BaseModel):
    type: OpType
    block_id: str | None      # ADD_NODE
    block_version: str | None # ADD_NODE  (default "1.0.0")
    node_id: str | None       # SET_PARAM / REMOVE_NODE / RUN_PREVIEW (logical id e.g. "n1")
    src_id: str | None        # CONNECT (logical)
    src_port: str | None      # CONNECT
    dst_id: str | None        # CONNECT
    dst_port: str | None      # CONNECT
    params: dict | None       # ADD_NODE (initial) / SET_PARAM ({"key":..., "value":...})
    # graph 執行階段填回:
    result_node_id: str | None = None    # add_node 真實產出的 id
    result_status: Literal["pending","ok","error"] = "pending"
    error_message: str | None = None
    repair_attempts: int = 0

Logical vs real node_id：LLM 在 plan 用邏輯 id 「n1, n2, n3」串連，graph 在 add_node 完成時把真實 id 填回 result_node_id，並把後續 op 的 node_id / src_id / dst_id 替換成真實 id。LLM 不需要知道真實 id 怎麼算。

⑤ BuildGraphState (LangGraph state schema)

class BuildGraphState(TypedDict):
    # 輸入（不變）
    instruction: str
    base_pipeline: dict | None       # PipelineJSON dump
    user_id: int

    # plan stage
    plan: list[dict]                 # list[Op] (model_dump)
    plan_validation_errors: list[str]
    plan_repair_attempts: int

    # confirm stage
    is_from_scratch: bool
    user_confirmed: bool | None      # None = 等待中

    # execute stage
    cursor: int                      # 目前執行到 plan[cursor]
    logical_to_real: dict[str, str]  # {"n1": "n3", "n2": "n4"} 邏輯 → 真實 id
    failed_op_idx: int | None        # 升級到 repair_plan 用

    # output
    final_pipeline: dict | None
    sse_events: list[dict]           # node 累積 SSE，graph runtime flush
    status: Literal["running","needs_confirm","plan_unfixable","cancelled","finished","failed"]
    summary: str | None

⑦ Confirm gate 的 SSE / interrupt 流程

┌─────────── server (sidecar) ───────────┐         ┌─────── client (前端) ────────┐
│                                         │         │                               │
│  [4] confirm_gate_node:                 │ SSE →   │ onPlanProposed(plan, summary) │
│     emit plan_proposed (JSON of plan)   │ ────►   │   render confirm card         │
│     emit confirm_pending                │         │                               │
│     interrupt()  ← graph 暫停           │         │ user 按 Apply                 │
│     state 寫進 LangGraph MemorySaver    │         │   POST /agent/build/{sid}/confirm
│                                         │ ◄────── │     {confirmed: true}         │
│  router 端 receive POST:                │         │                               │
│     graph.invoke(None, config={...sid…})│         │   reconnect SSE (or keep open)│
│     ← graph 從 interrupt 處 resume      │         │                               │
│     繼續 dispatch_op loop               │ SSE →   │ onOpDispatched / onOpCompleted│
│                                         │         │                               │
└─────────────────────────────────────────┘         └───────────────────────────────┘

Checkpointer 選擇：v1 用 LangGraph 內建 MemorySaver（in-process）。 Sidecar 是單 pod，重啟會掉；但目前 v1 行為已經是這樣（_PAUSED_SESSIONS 也是 in-memory），所以沒退化。 v2.x 升級成 Redis 時換 RedisSaver 即可，無需改 graph 程式碼。
