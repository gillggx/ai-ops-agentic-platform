## D-1 · FK reconcile (Java Long → Integer)

### Measured mismatch
- **Python DB schema** (live on EC2): 59 columns `integer` (4-byte INT), 6 columns `bigint`
- **Java entities** (`java-backend/src/main/java/com/aiops/api/domain/`):
  - 30 entities declare `private Long` for id / FK fields
  - 5 entities declare `private Integer`
- ⇒ Most Java fields would fail `ddl-auto=validate` against the live schema

### Decision (per user 2026-04-25): **option (a) — Java changes to Integer**
Rationale:
- Live schema is INT — no DB migration risk
- JPA `@Column` type is the source of mismatch; purely code-side fix
- Trade-off: caps IDs at 2^31 (~2.1B). Current row counts are well under 1M across every table — non-issue for foreseeable use

### The 30 entities (found via `grep private Long`)

```
AutoPatrolEntity ExecutionLogEntity SkillDefinitionEntity PipelineEntity
GeneratedEventEntity AlarmEntity RoutineCheckEntity McpDefinitionEntity
BlockEntity AuditLogEntity UserPreferenceEntity ScriptVersionEntity
PublishedSkillEntity PipelineRunEntity PipelineAutoCheckTriggerEntity
ItemEntity FeedbackLogEntity CronJobEntity CanvasOperationEntity
... and ~11 more
```

### Execution

**D-1a · Mechanical rewrite** (est 1h)
```bash
cd java-backend/src/main/java/com/aiops/api/domain
find . -name "*Entity.java" -exec sed -i '' \
  -e 's|private Long id;|private Integer id;|' \
  -e 's|private Long \([a-zA-Z]*\)Id;|private Integer \1Id;|' \
  {} +
```
This covers the mechanical 80% (any `private Long <something>Id`).

**D-1b · Audit stragglers**
Any `Long` that's NOT an id/FK column (e.g. `durationMs`, `tokenCount`) should stay Long. Re-grep to ensure only id fields were touched:
```bash
grep -rn "private Long" java-backend/src/main/java/com/aiops/api/domain
# Expected residual: durationMs, token counts, size counters, timestamps
```

**D-1c · Fix compilation breakage**
Getters/setters generated by Lombok auto-adjust. Controllers that pass `Long` around need manual update — grep `: Long `  and `<Long>` in `api/*` + `sidecar/*`:
```bash
grep -rn "Long\b" java-backend/src/main/java/com/aiops/api/api \
                  java-backend/src/main/java/com/aiops/api/sidecar
```
Each will need to change `Long → Integer` or explicit cast.

**D-1d · JDBC driver check**
PostgreSQL JDBC returns `INTEGER` column as `int`/`Integer`. No driver changes. Hibernate will map `@Column` `Integer` ↔ SQL `int` cleanly.

**D-1e · Run Spring tests**
```bash
cd java-backend
./gradlew test --no-daemon
```
Fix any compile errors. No DB migration needed.
