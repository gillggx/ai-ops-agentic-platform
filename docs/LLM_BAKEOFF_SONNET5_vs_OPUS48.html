D結論與建議

  • Opus 4.8 = 品質天花板：嚴格 16/17（唯一勝負手 xbar_r 做對）、最快、最果斷。代價是 direct+cache 實測約 GLM 的 20×成本。
  • Sonnet 5 被 GLM 支配：嚴格 15/17（與 KIMI 同兩個坑，xbar_r 還退化），direct+cache 成本仍是 GLM 的 ~5× —— 更貴又不更準，沒有升級理由。
  • weco_rules 這類失分換模型救不了：三個模型都中，屬系統層（knowledge / block doc）弱點，應從那裡修。
  • 決策：GLM-5.2 續任 prod 預設。Opus 僅在「難題（fleet / composite / 特定管制圖）人工升級」的 on-demand 場景才值得叫用，不當常駐 driver。
  • 若真要引入 Opus 升級層：務必走 direct Anthropic API 以恢復 prompt cache（direct $52 vs OpenRouter 無 cache $68.6），並限縮到少數難案,否則帳單失控。
