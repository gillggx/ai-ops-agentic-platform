Platform Overview

AIOps 智慧維運平台
架構總覽

Server-side 平台搭載 AI Agent、串接對應的 LLM model；
同時開放 MCP 讓 local 端的 Claude Cowork 連入。
所有動作皆為 Human-in-the-Loop — 不直接對線上系統動手。

① Server-side 平台

平台跑在 Server，Agent 內建於後端

使用者從瀏覽器操作，所有分析與 Agent 推理都在 server 端完成。

② Local Claude Cowork

能做什麼
探索 blocks、preview / validate / execute pipeline、建 skill、提出 rule 變更提案
關鍵設計
Cowork 只能「提案」，走的是跟平台一樣的 discover → preview → validate 流程， 不會繞過任何檢查。
★ 最重要的設計原則

沒有任何一條路徑會 直接對線上動作

AI 與 Cowork 都只能提案，
真正的變更一律由人在 GUI 確認後才執行。

AI / Cowork 產出提案 → 人在標準介面 Review → 人 確認 / 編輯 → 系統才實際套用
  • ✕
    LLM 生成的 block / skill 永遠是草稿 — 人 review + 編輯後才 commit。
  • ✕
    Cowork 提出的 rule 變更（啟用 / 停用 / 刪除）只能從已登入的 GUI 觸發，MCP 本身無法執行。
  • ✓
    危險動作與線上系統之間，永遠隔著一個人的確認 — 這是不可繞過的 gate。
