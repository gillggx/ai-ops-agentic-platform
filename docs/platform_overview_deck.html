★ 最重要的設計原則

兩條路徑都 不直接對線上動作 — Human-in-the-Loop

AI 與 Cowork 都只能提案，真正的變更一律由人在 GUI 確認後才執行。

AI / Cowork 產出提案 → 人在標準介面 Review → 人 確認 / 編輯 → 系統才實際套用
  • ✕
    LLM 生成的 block / skill 永遠是草稿 — 人 review + 編輯後才 commit。
  • ✕
    Cowork 提出的 rule 變更（啟用 / 停用 / 刪除）只能從已登入的 GUI 觸發，MCP 本身無法執行。
  • ✓
    危險動作與線上系統之間，永遠隔著一個人的確認 —— 這是不可繞過的 gate。

強大的 AI 自動化，但方向盤永遠握在人手上。
