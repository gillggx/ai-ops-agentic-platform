4. v30 vs v27 fork (2026-05-17 起 v30 為預設)

方面v27 (legacy macro_plan)v30 (current default ReAct)
Planning macro_plan_node 一次性決定 ops list (LLM 沒看到資料就要列 block) goal_plan_node 列 phases (intent-only)，每 phase 邊看資料邊決定 block
Build loop compile_chunk × N (per-op dispatch + autofix) agentic_phase_loop (per-phase ReAct, max 8 rounds)
Verify finalize dry-run + inspect_execution + reflect_plan phase_verifier (rule + B2 LLM-judge semantic)
Connect 智慧 無，LLM 自己猜 port (常錯) v30.12 matched-only CONNECT OPTIONS view (type-compatible only)
Catalog 兩邊共用 seed.py + 55 blocks，v30.11 加的 block_find 兩邊都看得到
觸發方式 之前 default v30.14 (2026-05-17) 已切為 default，emergency revert: AGENT_BUILD_V30=0 env

一旦 pipeline_json 完成（不管是 chat 還是 builder 產的，還是 skill 保存的）， 跑的時候三條路最終都會 hit PipelineExecutor + BUILTIN_EXECUTORS。 所以 block 行為 / runtime schema / 資料源都是共用一套。

Boot invariant Sidecar 啟動時 _boot_invariants.check_block_consistency 驗 4 個數字: BUILTIN_EXECUTORS / SIDECAR_NATIVE_BLOCKS / seed.py / DB pb_blocks 必須一致。 目前: 55 / 55 / 55 / 55。
