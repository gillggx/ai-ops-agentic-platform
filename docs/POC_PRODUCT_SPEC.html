Skill Library POC — 部署規格

對象：DevOps / Platform  ·  Branch poc/skill-library-20260624  ·  Repo gillggx/ai-ops-agentic-platform  ·  配套：POC_HANDOVER_PLAN.html  ·  2026-06-24

一句話：給半導體運維工程師用的 AI 技能庫 —— 工程師用自然語言或手動建「分析技能（資料 pipeline）」， LLM agent 把需求轉成 block pipeline、對外部資料源跑、回傳圖表 / 表格 / 判定。本 POC 只保留這條核心鏈， 拿掉內建 simulator 與 patrol / alarm / fleet；外部資料一律走可設定的 System MCP。

1範圍

In Scope保留

  • L1 Library：瀏覽 / 搜尋已發布 skill + block 目錄
  • L2 Authoring：自然語言（Glass Box agent）或手動建 skill
  • L3 Try-Run：草稿 skill 對 live 資料試跑、看結果
  • Block Docs：每個 block 的描述/參數/範例（LLM 與 UI 的唯一真相）
  • Build Trace：每次建構的 agent 推理軌跡（除錯用）
  • System MCP：admin 設定的外部 HTTP 資料源（含 ${ENV} 機密插值）

Out of Scope已移除

  • ontology_simulator 服務（及其 MongoDB）
  • auto-patrol / alarm / rules engine / scheduler（L4）
  • Topology / Fleet 儀表板（FleetSimulatorClient stub 成空）
  • /admin/simulator-health 及相關 simulator UI

4外部資料層 — System MCP

POC 的資料來源。admin 在 /admin/system-mcps 註冊：endpoint URL、HTTP method、HTTP headers、input schema。 Header 值支援 ${ENV_VAR} 插值 —— 機密不存 DB；runtime 由 resolve_headers() 從 sidecar 環境取代， 缺哪個 env 就明確報錯（INVALID_MCP_CONFIG）。

維運重點：MCP header 設定裡每個 ${NAME} 機密，都必須存在於 sidecar 的環境（python_ai_sidecar/.env），例如 EXTERNAL_API_TOKEN=...。

5部署（EC2 systemd · 現行）

systemd units 在 deploy/：aiops-app / aiops-java-api / aiops-java-scheduler / aiops-python-sidecar。

全新主機 
# clone POC branch
git clone -b poc/skill-library-20260624 \
  https://github.com/gillggx/ai-ops-agentic-platform.git /opt/aiops
cd /opt/aiops
# 填好各服務 .env 機密（見 §6），然後：

B
既有主機 / 重新部署 
cd /opt/aiops && git fetch && git checkout poc/skill-library-20260624 && git pull
# 若殘留 simulator unit，停掉：
sudo systemctl stop ontology-simulator 2>/dev/null; sudo systemctl disable ontology-simulator 2>/dev/null
sudo bash deploy/java-update.sh   # 重建 Java jars + sidecar venv，重啟 Java + sidecar
sudo bash deploy/update.sh        # 前端（本 branch 已無 simulator）
DB 初始化（重要）：
  • Migration 在 java-backend/.../db/migration/V*.sql；prod Flyway 關閉 → 新 V*.sql 要手動 psql -f（DB 機密在 java-backend/.env）。
  • Block 目錄：java-update.sh 會套 db/canonical/pb_blocks_canonical.sql（ON CONFLICT DO NOTHING）—— 全新 DB 補齊完整 56 block，既有 DB 為 no-op。

K8s（未來）：每服務各自 image、EXPOSE 8080、service port → 80；manifests 在 deploy/kubernetes/（已移除 simulator component）。run wrapper 待 K8s target 決定後補。
