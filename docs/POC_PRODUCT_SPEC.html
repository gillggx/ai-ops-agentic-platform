Skill Library POC — 功能規格

產品功能說明（What it does） ·  部署見 POC_DEPLOYMENT_SPEC.html  ·  接手見 POC_HANDOVER_PLAN.html  ·  2026-06-24

產品定位：給半導體運維工程師的「AI 分析技能庫」。工程師用自然語言提出運維問題， 平台的 LLM Agent 自動把它變成一條資料分析 pipeline、對資料源執行、回傳圖表 / 表格 / 判定； 好用的分析可發布成 Skill 給整隊重複使用。把「會接資料的少數人」的能力，變成每個工程師一句話就能取得。

核心功能（六大功能，各一專章） ① Library 技能庫② Authoring 編寫③ Try-Run 試跑 ④ Block Docs + Advisor⑤ Build Trace⑥ System MCP

A核心概念（領域語彙）

Skill 技能

已發布、可重複用的分析 pipeline，回答一類運維問題。

Block 積木

最小操作單元（取資料/轉換/畫圖/判定）。內建 56 個，各自帶完整文件。

Pipeline 管線

多 block 串成的有向圖 —— 一個 Skill 的實作。

System MCP 資料源

外部 HTTP 資料來源；機密以 ${ENV} 引用。

Agent 代理

把自然語言變 pipeline 的 LLM；會規劃、選 block、自我修正。

Knowledge 知識層

規劃提示 + 領域模型 + block 文件，讓 Agent 接地。

①功能一 · Library 技能庫ON_DUTY+

做什麼：可瀏覽 / 搜尋的「已發布 Skill + Block 目錄」入口（/skills）。讓不會建構的人也能直接取用現成分析。

子功能說明
Skill 瀏覽 / 搜尋列出所有已發布 Skill，依名稱 / 用途搜尋、依類別篩選
Skill 詳情頁/skills/[slug]：看用途、需填的輸入參數、輸出型態（圖/表/判定）
直接執行選 Skill → 填參數 → 一鍵跑出結果（不進建構流程）
Block 目錄瀏覽56 block 依類別（source / transform / output / logic / check）瀏覽
權限分級ON_DUTY 可用已發布 Skill；PE / IT_ADMIN 另可進入編寫

②功能二 · Authoring 技能編寫（兩種模式）PE+

做什麼：把需求變成一個 Skill 草稿。提供「自然語言（Glass Box）」與「手動畫布」兩條路（/skills/new、/admin/pipeline-builder）。

A · Glass Box（自然語言建構）

子功能說明
自然語言下指令一句話描述需求，Agent 自動規劃 + 建構
意圖澄清需求模糊時主動反問（選項卡），不亂猜
計畫確認建構前先出「要達成的階段（phases）」確認卡，使用者按確認才開建
透明建構即時顯示每步在加哪個 block、為什麼（Glass Box）
自我修正結構或參數出錯當下自動偵測並修（懸空節點、型別錯誤、provider 抖動重試）

B · Manual Builder（手動畫布）

子功能說明
畫布拼接自己加 / 連 / 設定 block，完全掌控
Block Docs 輔助設定時可開該 block 的文件抽屜看參數 / 範例
即時結構驗證接線 / 參數錯誤即時提示（缺必填、欄位不存在、孤兒節點…）

C · 共通

子功能說明
Skill 中繼資料命名、用途、宣告輸入參數（讓使用者之後填）
觸發模式once（單次）/ schedule（排程）/ patrol（巡檢）等綁定方式
草稿 → 發布編輯 / 反覆迭代；確認後發布給團隊

③功能三 · Try-Run 試跑PE+

做什麼：把草稿 Skill 對真實資料跑一遍、當場看結果，確認對了再發布。

子功能說明
live 執行對 System MCP 連到的真實資料源執行 pipeline
結果渲染依輸出型態呈現：圖表 / 表格 / 單一數值 / 判定（過或不過）
結果分頁（Lite Canvas）建構畫面內直接看「結果」分頁，不必離開
錯誤明確化跑失敗給具體碼與提示（COLUMN_NOT_FOUND、MCP_UNREACHABLE、參數型別…）
發布前把關沒試跑成功就不建議發布

④功能四 · Block Docs + Block AdvisorPE+

做什麼：自帶文件的 block 目錄，加上一個會回答「這 block 怎麼用」的問答助手（/admin/block-docs）。

子功能說明
每 block 文件用途 / 參數 schema / 範例 ——「唯一真相來源」，Agent 與 UI 讀同一份，不會人/機看到不同版本
Docs 抽屜建構時隨手開，看某 block 怎麼填、輸出什麼
Block Advisor 問答問「這 block 怎麼用 / A 跟 B 差在哪 / 我該用哪個」→ 自動分類意圖後回答（EXPLAIN / COMPARE / RECOMMEND）
目錄規模56 block：source 4 · transform 20 · output 23 · logic 8 · check 1
18+ 圖表型態line / bar / scatter / box / histogram / pareto / X-bar/R 管制圖+WECO / IMR / EWMA-CUSUM 漂移 / Q-Q 常態 / parallel-coords / wafer heatmap / defect map …

⑤功能五 · Build Trace 建構軌跡ADMIN

做什麼：把每次建構的「Agent 怎麼想、怎麼做」完整留存，供回看與除錯（/admin/build-traces）。是「信任 + 可維護」的關鍵。

子功能說明
完整軌跡記錄 plan、每次 LLM 呼叫、每個 graph 步驟、驗證判定、finish_reason
軌跡列表後台列出歷次建構，可點入單筆
Summary 視圖結構化呈現 plan → 卡在哪個 phase → 該 phase 每輪歷程
除錯用途回答「它為什麼這樣建 / 卡哪 / 選錯哪個 block」，不用瞎猜
可重現配 trace replay 受控重跑某決策點（測「改 X 會不會改 LLM 的選擇」）

⑥功能六 · System MCP 外部資料源ADMIN

做什麼：POC 的資料連接層 —— admin 把外部 HTTP 服務註冊成可被 Skill 取用的資料源（/admin/system-mcps）。

子功能說明
來源設定endpoint URL、HTTP method、input schema（參數定義）
HTTP Headers + 機密headers 支援 ${ENV_VAR} 插值；機密不存 DB，runtime 從 sidecar 環境取代，缺值明確報錯
自帶文件MCP description 必含用途 / 回傳欄位 / 語義 —— Agent 只看 description 就會用
連動衍生（V54）勾選「連動產生」可自動生對應的 Data Block + Skill 草稿（LLM 生，須 review 後才 commit）
執行Skill 跑到該資料源時走 MCP proxy dispatch 取數
維運重點：MCP header 引用的每個 ${NAME} 機密，都須存在於 python_ai_sidecar/.env。

CAgent 怎麼把「一句話」變 pipeline（貫穿 ②③）

理解意圖→（模糊則反問）→ 規劃 phases→確認→ 逐步建構＋自檢→執行→呈現
環節功能語意
理解意圖判斷是「查資料 / 比較 / 排名 / 概念問答」；不足則主動反問
規劃拆成「想達到的階段性結果」，先講意圖、不先綁工具
建構＋自檢逐 phase 選 block、設參數；每步結構檢查（懸空節點當下抓），確保跑得動
穩定性provider 抖動自動重試、參數型別偶錯自動修正 —— 一次小失誤不毀掉整個建構

兩入口：Chat 面板（運維對話問答）與 Builder Glass Box（建構技能）。流程控制走確定性 graph node，行為可測、可回歸。

D角色與權限

角色能做什麼
IT_ADMIN全部 + 使用者/角色管理、System MCP 設定、Build Trace 後台
PE（製程工程師）編寫 / 建構 / 發布 Skill、設定、跑分析
ON_DUTY（值班）使用已發布 Skill、讀資料；不可建構新 pipeline / 寫全域記憶
待決策（產品）：值班問臨時數據問題時，chat 目前導去「建 pipeline」，但值班無建構權 → 拿不到答案。是否改走「直接答 / 跑已發布 skill」待 stakeholder 拍板。
