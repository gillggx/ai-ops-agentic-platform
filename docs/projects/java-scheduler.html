java-scheduler

Phase 2 拆出來的獨立 Spring Boot 程序。所有 @Scheduled 跑這 — event polling、skill auto-fire、cron jobs，跟 user-facing API 分家避免 GC 干擾。

Port8003
TechJava 17 · Spring Boot 3.5.14 · Redis 7 (distributed lock)
DBshares aiops_db with java-backend (read-only for skill / patrol; writes own audit + alarm)
Service unitaiops-java-scheduler.service

主要服務 (@Scheduled jobs)

EventPollerService

simulator → generated_events → dispatch

  • 每 30s 拉 simulator /events watermark
  • OOC / patrol_check events 寫 generated_events table
  • 呼叫 EventDispatchService.dispatchGeneratedEvent

EventDispatchService

event → fan-out

  • Legacy: auto_patrols (空表)
  • v6.1 new: dispatchToSkillsByEvent — 查 skill_documents WHERE trigger_config.type=event AND event=<name>
  • 呼叫 SkillApiClient.dispatchSkill(slug, "system_event")
  • per-skill 5min Redis lock 防 schedule+event 同時 fire

SkillScheduleService v6.1

cron skills 自動跑

  • @Scheduled fixedDelay=60s, initialDelay=30s
  • scan stable skills with trigger_config.type=schedule
  • isDue = last_system_triggered_at + interval (hourly/daily) ≤ now
  • fire via SkillApiClient.dispatchSkill(slug, "system_schedule")

AutoPatrolSchedulerService

legacy patrol scheduler

  • dynamic CronTrigger registration per auto_patrol
  • schedule mode + once mode
  • auto_patrols table 目前空 — Phase 11 v6 migration 把模型搬到 skill_documents 了

SkillApiClient v6.1

HTTP client

  • POST java-backend /internal/skills/by-slug/{slug}/run-system
  • X-Internal-Token (env JAVA_INTERNAL_TOKEN)
  • fire-and-forget — java-backend 同步回 202 ack 再 async 跑 skill

DistributedLockService

Redis-based mutual exclusion

  • Phase 3 — multi-pod scheduler safety
  • runWithLock(key, ttl, runnable) — Redis SETNX
  • fail-open: Redis down → run anyway + warn (env AIOPS_LOCK_FAIL_OPEN=true)

v6.1 migration: skill trigger 補洞

Phase 11 v6 把 trigger model 從 auto_patrols 搬到 skill_documents.trigger_config。但 java-scheduler 當時沒同步改 — 永遠不知道 skill 有 trigger，alarm 3 天斷流。v6.1 ship 兩個新 service (SkillScheduleService + EventDispatchService.dispatchToSkillsByEvent) 補上。

java-scheduler ── @Scheduled ──┬── SkillScheduleService.tick() 每 60s │ ↓ │ scan skill_documents (stable + schedule mode) │ ↓ │ isDue? → SkillApiClient.dispatchSkill(slug, "system_schedule") │ └── EventPollerService 每 30s ↓ new OOC events → EventDispatchService ├── auto_patrols (legacy, empty) └── skill_documents (v6.1 new) ↓ SkillApiClient.dispatchSkill(slug, "system_event")

Trigger config schema

typeshape例子
schedule{type:"schedule", schedule:{mode:"hourly"|"daily", every:1}}每小時自動跑
event{type:"event", event:"OOC", target:{kind:"all"}}每收到 OOC event 跑一次
user{type:"user"}不自動跑，user UI 按按鈕觸發
