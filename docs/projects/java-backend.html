java-backend

唯一 DB owner。所有 CRUD、Auth、Skill / Pipeline / Alarm registry 都住這。一個 Spring Boot 程序 (port 8002)，所有 user-facing API + 給 sidecar 跟 scheduler 用的 internal API 都在這。

Port8002
TechJava 17 Temurin · Spring Boot 3.5.14 · Maven · PostgreSQL 16 + pgvector
Service unitaiops-java-api.service
Deploydeploy/java-update.sh (mvn package + restart)

API surface

PrefixAuth用途
/api/v1/*JWT (user / admin / PE / IT_ADMIN)frontend user-facing CRUD: skills, pipelines, alarms, users
/internal/*X-Internal-Token (JAVA_INTERNAL_TOKEN env)service-to-service: blocks, mcps, agent-knowledge, skills/by-slug/{slug}/run-system, alarms, generated-events
/auth/*open (login) + JWT (refresh)NextAuth-style local credentials + OIDC (multi-provider)
/actuator/*openhealth probes

主要模組

domain/

JPA entities + repos · shared with java-scheduler

  • pipeline/ — BlockEntity, PipelineEntity, PipelineAutoCheckTriggerEntity
  • skill/ — SkillDocumentEntity, SkillDefinitionEntity, SkillRunEntity, PublishedSkillEntity
  • alarm/ — AlarmEntity (trigger_event, severity, status, ack/dispose)
  • blockdoc/ — BlockDocEntity (admin-editable Markdown)
  • event/ — EventTypeEntity, GeneratedEventEntity
  • patrol/ — AutoPatrolEntity (legacy, kept for back-compat)
  • user/ — UserEntity + RoleChangeLogEntity

api/v1/ + api/internal/

REST controllers

  • SkillDocumentController — POST /{slug}/run (SSE, user-triggered)
  • InternalSkillController — POST /by-slug/{slug}/run-system (service-triggered, v6.1)
  • InternalBlockController — GET /internal/blocks (sidecar reads catalog)
  • InternalBlockDocController — GET/PUT block_docs Markdown
  • AlarmController — CRUD + ack/dispose
  • UserController — admin user management

api/skill/SkillRunnerService

Skill 執行引擎

  • 讀 skill_documents.steps → 對每個 step pipeline_id 呼叫 sidecar /internal/pipeline/execute
  • 寫 skill_runs (triggered_by: user / user_test / manual / system_schedule / system_event)
  • block_alert 觸發時呼叫 InternalAlarmController.create → 寫 alarms table
  • SSE stream events 給 controller bridge 到 frontend

auth/

JWT + Internal Token filter

  • JwtAuthFilter — 解析 Bearer JWT → AuthPrincipal
  • InternalServiceTokenFilter — X-Internal-Token 比對 aiops.internal.token
  • Role hierarchy: IT_ADMIN > PE > ON_DUTY
  • OIDC plug-in (Google / Microsoft / custom) — env-gated

Flyway migrations

在 src/main/resources/db/migration/。生產 Flyway 停用 (spring.flyway.enabled=false) — 新 V** SQL 不自動跑，要 SSH EC2 手動 psql -f。Dev / CI Flyway 開啟。

Prod migration gotcha: 加新 Flyway file 後不會自動 apply。要 sudo -u postgres psql aiops_db -f V49__xxx.sql 手動跑。DB pwd 在 /opt/aiops/java-backend/.env。

關鍵環境變數

DB_URLjdbc:postgresql://localhost:5432/aiops_db

JAVA_INTERNAL_TOKENX-Internal-Token 比對值 (sidecar + scheduler 用這個 call /internal/*)
JWT_SECRETJWT signing
OIDC_*OIDC provider config (optional, env-gated)
PYTHON_SIDECAR_URLhttp://127.0.0.1:8050
PYTHON_SIDECAR_TOKENX-Service-Token sidecar 比對值
