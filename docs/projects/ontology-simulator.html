ontology_simulator

合成製程資料源。模擬半導體製程的 lot / tool / step / SPC / APC / FDC / EC events，提供跟 production ontology 完全相同的 API 介面。Agent 跑 process_history 拉的就是這。

Port8012
TechPython 3.11 · FastAPI · MongoDB · NATS (event bus, optional)
Runvia venv_ontology on EC2 (no systemd unit, started via script)

API surface

Endpoint用途
GET /api/v1/tools列出所有設備 (EQP-01 .. EQP-10)
GET /api/v1/eventsprocess events stream — 支援 since_id / limit / start_time
GET /api/v1/process/infoprocess events by tool_id / lot_id / step (snake_case + camelCase aliases)
GET /api/v1/process/events同上 alternative shape
GET /api/v1/process/history歷史記錄 (sidecar block_process_history 走這)
POST /api/v1/inject-anomaly注入 OOC / drift 異常 (測試用)

資料 shape

每個 process event 約這樣（nested=true 時拉完整版）:

{
  "eventTime": "2026-05-20T10:00:00",
  "toolID": "EQP-01",
  "lotID": "LOT-0935",
  "step": "STEP_009",
  "spc_status": "PASS" | "OOC",
  "fdc_classification": null | "...",
  "spc_charts": [           // 12 chart types per event
    {"name": "xbar_chart", "value": 14.95, "ucl": 17.5, "lcl": 12.4, "is_ooc": false, "status": "PASS"},
    {"name": "r_chart", ...},
    {"name": "cusum_chart", ...},
    ...
  ],
  "APC": { ... 20+ APC parameters ... },
  "DC": { ... 15+ chamber sensors ... },
  "RECIPE": { ... recipe metadata ... },
  "FDC": { ... fault detection ... },
  "EC": { ... event chain ... }
}

EventPoller 對接

java-scheduler 的 EventPollerService 每 30s 拉 /api/v1/events?since_id=X 拿新 events，寫 generated_events table，再 dispatch 到 skills (event mode)。

角色定位

純資料服務: simulator 不知道 Agent 的存在，跟它對話的 API 介面跟 production ontology 完全相同。換到 real factory 只是改 base URL，business code 不動。

啟動 (EC2)

cd /opt/aiops/ontology_simulator
source venv_ontology/bin/activate
uvicorn app.main:app --host 127.0.0.1 --port 8012 &

# Health
curl http://localhost:8012/health
# → {"status": "ok"}

Repo path

程式碼在 ontology_simulator/（跟 fastapi_backend_refactored 同 root）。EC2 path /opt/aiops/ontology_simulator。
