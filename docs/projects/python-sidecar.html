python_ai_sidecar

所有 AI agent + 55 個 block executor + pipeline runner 都住這。LangGraph 跑 build / chat / advisor; Pipeline Executor 跑使用者 Save 後的 pipeline。

Port8050
TechPython 3.11 · FastAPI · LangGraph · Anthropic SDK
DBnone 不直接連 PostgreSQL — 全走 JavaAPIClient → java-backend
Service unitaiops-python-sidecar.service
Deploydeploy/java-update.sh (rebuild venv + restart)

主要模組

agent_builder/

build agent · advisor

  • graph_build/ — LangGraph state machine (goal_plan + agentic_phase_loop + phase_verifier)
  • advisor_v2/ — tool-using doc Q&A loop (v6.2)
  • advisor/ — legacy graph-based advisor (KNOWLEDGE / AMBIGUOUS fast path)
  • tools.py — BuilderToolset (~25 tools for ReAct agent)
  • session.py — AgentBuilderSession + StreamEvent

agent_orchestrator_v2/

chat panel orchestrator

  • 5-bucket intent classifier
  • tool_call / quick_fact / knowledge / pb_build / clarify routing
  • Calls Java skill endpoints when LLM picks a skill

pipeline_builder/

55 block executors + validator

  • blocks/ — 1 file per block (filter, unnest, sort, step_check, ewma_cusum, alert, ...)
  • seed.py — block_definitions catalog (synced to Java DB at boot)
  • validator.py — C1-C16 structural rules (covers gate, param schema, orphan, placeholder leak, ...)
  • seedless_registry.py — pulls block catalog from java-backend on startup
  • pipeline_schema.py — Pydantic models for PipelineJSON / Node / Edge

clients/

talks to Java

  • java_client.py — async HTTP client for all /internal/* endpoints on java-backend
  • X-Internal-Token auth (env PYTHON_SIDECAR_TOKEN)

routers/

FastAPI surfaces

  • /internal/agent/build — Glass Box builder (SSE stream)
  • /internal/agent/chat — chat panel
  • /internal/pipeline/execute — pipeline runner (Java calls this for skill runs)
  • /internal/agent/build/traces/* — trace inspection + summary

background/

lifespan tasks

  • event_poller.py — disabled by default; java-scheduler took over polling
  • embedding_backfill.py — backfill missing embeddings for RAG
  • nats.py — optional NATS subscriber (disabled by default)

啟動行為

  1. Boot — load env, init httpx clients
  2. SeedlessBlockRegistry: GET java-backend /internal/blocks → cache 55 block specs in memory
  3. Block invariant check: 55 builtin = 55 native executor = 55 in seed = 55 in DB
  4. Background tasks: embedding_backfill (always), event_poller (env-gated, default off), NATS subscriber (env-gated)
  5. FastAPI ready on :8050

Deploy lifecycle

local commit → push origin main
  ↓
ssh ec2 'cd /opt/aiops && sudo git pull'
  ↓
sudo ./deploy/java-update.sh
  ↓  (rebuilds venv + Java jars + restarts both services)
sidecar restart at boot:
  - re-loads seed.py
  - re-syncs block catalog to java-backend DB
  - block invariant check
  - ready
Common gotcha: deploy/update.sh only restarts aiops-app + simulator. Sidecar changes need deploy/java-update.sh OR explicit sudo systemctl restart aiops-python-sidecar.
