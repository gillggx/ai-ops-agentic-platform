4待你拍板

Q1 · 先做哪個視圖?　建議視圖①(單 build 時間軸)先做 —— 它最能「秀」agent 行為,且一次 build 的資料量小、渲染簡單。
Q2 · 橫軸用 phase 進程 還是真實時間?　建議 phase 進程(真實時間受 provider 延遲扭曲,不好看)。
Q3 · 放哪裡?　既有 /admin/build-traces 頁加一個「Agent 視圖」分頁,還是獨立新頁 /agent-activity?
Q4 · 記憶讀取要畫多細?　v1 只標「讀了 N 類」,還是列出具體召回哪幾條(要多一個 read-log,成本較高)?
