1先看 Claude Code（我）自己怎麼「學」

關鍵事實:我不更新權重。我的「學習」全部外部化到 harness 的檔案 — 這跟你們的 agent 一樣(你們也不 fine-tune),所以這個範本可以直接搬。

我的「記憶」= 磁碟上的 memory/*.md(每檔一個 fact,frontmatter 分類 user/feedback/project/reference)+ CLAUDE.md(常駐憲章)+ skills(可重用程序)。 被糾正時,我寫一條 feedback 記憶(帶 Why + How to apply),下次靠 description 召回。 「學習」= 寫進一個分類好、可召回的記憶庫,不是改權重。你們的 Supervisor 要做的就是這件事。

3Memory 怎麼分類（我的意見）

6 個 memory class × 3 個正交軸。軸的部分你們 agent_knowledge 幾乎都有了 — 只要再加一個 class 維度、每個 agent 讀寫自己那片。

6 個 Memory Class

domain領域知識(SPC=站級、APC=recipe 級、欄位語義)
preferenceuser 偏好(愛用哪些 block、精簡 vs 詳細、命名習慣)
presentation什麼資料 → 什麼呈現(時序→line、分佈→box、排名→bar)
correction被糾正學到的規則(帶 why + how-to-apply)= 我的 feedback 型
episodic過往 build 的成敗歷程(原始經驗,供 Supervisor 蒸餾)
procedure可重用的建構配方(≈ skill)

3 個正交軸（你們已有）

scopeglobal / user / skill / tool / recipe　已有 scope_type
layerplan / execute　已有 applies_to(V58)
優先級priority + always_on(直推 vs RAG)　已有
結論:不用重造 memory 基建。加一個 class 欄位到 agent_knowledge,搭配既有 scope/layer/priority,就是一套完整的分類記憶庫。

4兩種學習速度:Fast Path vs 蒸餾

Fast Path — in-session（即時)

Planner 收到 user feedback → 當下就寫一條 preference / correction(草稿) + 立刻重規劃。就像我在 session 中被糾正、馬上寫 memory 並改做法。特性:快、局部、未去重。

Slow Path — Supervisor 蒸餾（週期)

Supervisor 定期把大量 episodic traces 蒸餾成乾淨、去重、分類的 durable memory,並寫回。就是「反思/固化」。Supervisor = memory consolidator —— 把噪音經驗變成長期知識,還能發現「這是 doc 不齊」。

6修正後的互動迴路（依你的描述）

跟第一版線性圖不同:Planner 是 hub + 裁判,Builder 執行、結果回交 Planner 判對錯;Repair 吃「原 prompt + 現結果 + user feedback」三路輸入。

7幾個關鍵判斷

  1. Feedback intake 是唯一真正要新建的機制。其餘 9 個構件你們或多或少有了。這個機制 = 「user/Planner 的糾正 → 蒸餾成一條帶 why 的 correction 記憶」,就是我被糾正時做的事。沒有它,agent 每次都從零開始。
  2. Planner 同時是需求擁有者與裁判。它出需求給 Builder、又判 Builder 結果對錯 —— 這個「裁判」動作本身要留成 episodic,因為它是最有價值的訓練訊號(Supervisor 蒸餾的主料)。
  3. Repair 的三路對齊(原意圖 vs 現結果 vs feedback)就是它的 harness 核心。它學到的「原本哪裡錯」要寫成 correction,讓 Planner/Builder 下次避開 —— 這是 agent 之間透過記憶底層的間接學習。
  4. memory 一律「草稿 → 審核 → 版本化」。比照我寫 memory 前會查重、會刪錯的紀律,也比照你們 V54 衍生草案。自動寫記憶不設閘 = feedback 漂移。
  5. 短期 context ≠ 長期 memory,別混。working context(當前 build 的 observation)用完即棄;只有被蒸餾/被 feedback 認證的才升級成 durable memory。這是我 compaction 與 memory 的分界。
