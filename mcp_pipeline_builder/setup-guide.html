AIOps Pipeline Builder · 設定指南

讓你的 Claude 會用 AIOps

要讓 Claude（Desktop 或 Web）會替你建 AIOps pipeline，先選下面**兩條路其中一條**設好。第一次設定約 3–5 分鐘。

建議

路徑 A — 用 Connection

Claude Desktop 直接連到 AIOps 後端的 MCP server。Claude 會「真的看得到」你的 block 目錄、能 preview / execute / 存 pipeline。

適合：個人電腦、Mac/Windows、可以裝 Claude Desktop、可以連外網的環境。
備案

路徑 B — 用 Skill

把 skill 說明貼進 Claude.ai 的 Project / 對話 instructions。Claude 不會「真的呼叫」AIOps，但會照規則格式吐 pipeline JSON 給你貼回工具。

適合：公司電腦、組織不允許裝 MCP / connection、只能用 claude.ai web。

路徑 A · Connection 設定（Claude Desktop）

Claude Desktop → Settings → Connectors → Add custom connector。把下面 URL 貼進去即可。

  1. 開啟 Claude Desktop → 左下角頭像 → Settings
    macOS：⌘, ；Windows：Ctrl+,
  2. 切到 Connectors 分頁 → 點 Add custom connector
    如果沒有 Connectors 分頁，請先把 Claude Desktop 更新到最新版本。
  3. 填寫
    Name: aiops-pipeline-builder
    Remote MCP server URL:
    機密 https://aiops-gill.com/mcp-YOUR-SECRET-PATH/mcp
    注意這條 URL 含密鑰，等同於登入密碼。不要貼到公開 chat / 社群媒體 / 截圖外流。
  4. 儲存 → 回到任一聊天視窗
    右下角工具欄會多出一個 ✦ aiops-pipeline-builder 連結器。確認亮著（enabled）。
  5. 把這段話也貼到 Project custom instructions（讓 Claude 知道怎麼用）
    連線只給「能力」，但 Claude 要知道**順序**才不會瞎跑。下面 Path B 的 Skill 內容，無論你走 A 或 B 都應該貼一份到 Project custom instructions。

路徑 B · Skill 設定（claude.ai Project / chat instructions）

如果你的電腦不能裝 Connector，或是組織限制了 MCP，就用這條：把下面整段 Skill 內容貼進 claude.ai 的 Project custom instructions。

  1. 在 claude.ai 建一個 Project，命名為 AIOps Pipeline Builder
    如果不建 Project，也可以開新對話，把同樣內容貼在第一則 message 開頭。
  2. 打開 Project → Set custom instructions（或對話最頂端）
    把下面整段 Skill 內容（從 ## What you can do 開始到結尾）貼進去。
  3. 儲存 → 在這個 Project 內聊天即可
    Claude 不會真的呼叫工具，但會吐出格式正確的 pipeline JSON。你可以複製貼回 AIOps Pipeline Builder UI 的「import JSON」按鈕（如果有）或交給工程師。
提示就算你走路徑 A，這段 Skill 內容也一定要貼到 Project — Connection 給工具能力，Skill 給「使用順序」。

Skill 內容（複製整段貼進去）

# Skill — Build an AIOps analysis pipeline

## What you can do
You build a semiconductor-ops analysis **pipeline** — a small DAG of typed
**blocks** (source → transform → chart) — then run and save it. A saved pipeline
gets a `/pipeline-view/<id>` URL the human can open (DAG + rendered chart, and an
"open in Builder" link).

## Tools (from the `aiops-pipeline-builder` connector)
| tool | use |
|---|---|
| `list_blocks(category?)` | see available blocks |
| `explain_block(name)` | a block's real params/description/examples — the ONLY source of truth |
| `preview(pipeline_json, node_id)` | run up to a node, get its rows + columns (learn the data) |
| `validate(pipeline_json)` | structural check |
| `execute(pipeline_json)` | run the whole thing, get status + chart |
| `save_pipeline(name, pipeline_json, description?)` | save draft → returns id + view URL |

## The pipeline JSON you assemble
```json
{"version":"1.0","name":"...","inputs":[],
 "nodes":[{"id":"n1","block_id":"block_process_history","block_version":"1.0.0","params":{}}],
 "edges":[{"id":"e1","from":{"node":"n1","port":"data"},"to":{"node":"n2","port":"data"}}]}
```
Node positions are not needed — the UI lays out the DAG.

## Workflow — always in this order
1. `list_blocks()`, then `explain_block(name)` for each block you intend to use.
   **Never guess params** — read them from `explain_block`.
2. Add the **source** node first and `preview` it: confirm `rows > 0` and learn the
   **actual column names** before wiring anything downstream.
3. Build the rest using the columns you saw; `preview` each new node.
4. `validate`, fix any errors.
5. `execute`, confirm `status=success` and a chart/table came out.
6. `save_pipeline` and give the human the returned **view_url**.

## Gotchas (learned the hard way)
- `block_process_history` defaults `time_range="24h"`; sim data may be older →
  a 24h window returns 0 rows. If `preview` shows `rows=0`, widen `time_range`
  (`"7d"`/`"30d"`).
- **"all machines / 全廠"** → `block_list_objects(kind='tool')` →
  `block_mcp_foreach` → `block_unnest`, **not** a single `process_history`
  (that's one machine only).
- Multiple ids → do **not** comma-pack `tool_id`; leave it unset and filter
  downstream with `block_filter` `operator='in' value=[...]`.

驗證設好了沒

問 Claude 下面 4 句話，照順序成功 = 完成。

✓
「列出所有可用的 block」
路徑 A：Claude 會列出 30+ 個 block_*；路徑 B：Claude 會回說「我會用 list_blocks() — 請問你要看哪個 category？」
✓
「explain block_process_history」
A：回 block 的真實 description / params / examples（含 nested 預設 true）。B：Claude 會吐 explain_block 的 stub，提醒你貼回去。
✓
「幫我看 EQP-10 最近 7 天的 SPC 趨勢」
A：Claude 會 preview n1、看到 rows、加 chart、execute、save、給 /pipeline-view/<id> URL。B：Claude 會吐完整 pipeline JSON，可貼回 builder import。
✓
「跑全廠對比」
兩條路都應該用 list_objects → mcp_foreach → unnest 的模式，不是單 process_history。如果它直接用單個 tool_id，提醒它讀 Skill 的 Gotchas。

常見問題

我看不到 Connectors 分頁怎麼辦？
Claude Desktop 版本太舊。到 claude.com/download 拉最新版本。如果是組織管控不能更新，走路徑 B。
路徑 A 連上了但 Claude 還是不會用 — 怎麼辦？
Connection 只給「能力」，不給「使用方法」。把路徑 B 的 Skill 整段內容貼到同一個 Project 的 custom instructions。連線 + Skill 兩個都有才會穩。
路徑 B 吐的 JSON 怎麼貼回 AIOps？
打開 https://aiops-gill.com/admin/pipeline-builder，右上角有 Import JSON 按鈕。或是把 JSON 直接傳給有 connection 的同事，請他幫忙 save。
那條 URL 看起來像密碼 — 對嗎？
是。URL 路徑裡那串 hash 是 secret-path bearer，任何人有這條 URL 就能呼叫所有 builder 工具。不要外流。要 rotate 跟系統工程師說一聲。
能不能在手機 Claude.ai 用？
手機 app 目前不支援自訂 connector（路徑 A 不行）。路徑 B 可以 — Project custom instructions 在手機也讀得到。Claude 會吐 JSON，你截圖傳給有桌機的同事處理。
